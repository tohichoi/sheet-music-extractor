
# 🎵 악보 키프레임 추출기 (Sheet Music Keyframe Extractor)

[English Version](README.md) · [영어로 보기](README.md)

연주 영상에서 인쇄하기 딱 좋은 깔끔한 악보를 자동으로 추출해 보세요! 일시 정지 버튼을 누르느라 연습 흐름이 끊기는 걸 싫어하는 뮤지션들을 위해 빠르고 안전하게(로컬 환경) 설계되었습니다.

## 📖 프로젝트 개요

멋진 연주 영상을 보면 연습하고 싶은 의욕이 솟아나야지, 악보를 보느라 답답해하면 안 되겠죠? 이 프로젝트는 동영상에서 악보 화면만 쏙쏙 뽑아내어 A4 크기에 딱 맞는 깔끔한 PDF로 만들어줍니다. 이제 온전히 연주에만 집중해 보세요!

## ✨ 주요 기능

* 시각적인 방해물을 없애기 위해 악보 영역을 자동으로 감지하고 잘라냅니다.
* 마우스 드래그로 원하는 영역(ROI)만 선택하거나 끄고 켤 수 있는 인터랙티브한 기능을 제공합니다.
* OpenCV 전처리와 AI/휴리스틱 기법을 결합하여, 페이지가 넘어가는 순간을 정확히 포착하고 오작동을 줄인 강력한 이미지 처리 파이프라인을 갖추고 있습니다.
* 영상 분석이 오래 걸려도 화면이 멈추지 않도록 백그라운드에서 작업을 처리합니다.
* 추출된 이미지 중 원하는 것만 골라 순서를 정하고, 여백을 조절하여 A4용 PDF로 바로 내보낼 수 있습니다.
* MD5 기반의 해시 및 캐싱 기술로 이미 업로드했던 영상을 중복 처리하지 않아 시간을 절약해 줍니다.

## ✨ 장점 및 특징

이 프로젝트는 강력한 백엔드 처리와 직관적인 프론트엔드 경험을 분리하여, 영상에서 정확하고 빠르며 편리하게 악보를 추출하고 싶은 분들에게 최적화되어 있습니다.

### 백엔드의 장점 (Backend)

* **자동 크롭 & 똑똑한 영역(ROI) 추출:** 악보 영역을 자동으로 감지해 잘라냅니다. 사용자가 직접 영역을 지정해 원하는 부분만 추출할 수도 있으며, 드래그 선택 및 켜기/끄기 토글 기능을 지원합니다.
* **고급 이미지 처리 & AI 기술:** OpenCV 기반의 노이즈 제거와 AI/휴리스틱 기법을 결합하여 진짜 페이지가 넘어가는 순간을 찾아냅니다. 화질이 낮은 영상에서도 안정적으로 작동합니다.
* **사용자 맞춤형 PDF 생성:** 추출된 이미지 중 PDF에 포함할 이미지를 직접 고르고, 순서와 여백을 자유롭게 조절하여 완벽한 인쇄용 A4 PDF를 만들 수 있습니다.
* **백그라운드 중심 아키텍처:** 무거운 영상 프레임 분석과 추출 작업은 FastAPI의 BackgroundTasks로 실행되어, 작업 중에도 사용자 UI가 부드럽게 반응합니다.
* **스마트한 중복 제거 & 캐싱:** MD5 해싱과 로컬 캐시를 활용해 똑같은 영상을 다시 분석하는 낭비를 막고 컴퓨팅 자원을 아껴줍니다.
* **효율성 & 뛰어난 확장성:** 최적화된 OpenCV 처리, 프레임 스킵, 가벼운 DB 캐싱으로 로컬에서도 엄청나게 빠른 속도를 보여주며, 필요 시 Redis, Celery, Postgres 등으로 쉽게 확장할 수 있습니다.
* **안정성 & 견고함:** 임시 파일 및 캐시 관리, 재시도 전략, 안정적인 키프레임 저장 기능으로 오랜 시간이 걸리는 작업도 끄떡없습니다.

### 프론트엔드의 장점 (Frontend)

* **실시간 비디오 재생 & 미리보기:** 웹에서 영상을 바로 재생하고, 악보 추출 진행 상황과 썸네일이 실시간으로 업데이트되는 것을 확인할 수 있습니다.
* **인터랙티브 영역(ROI) 선택:** 마우스로 드래그해서 악보 영역만 지정하거나 해제하는 것을 직관적으로 할 수 있습니다.
* **PDF 내보내기를 위한 이미지 선택 UI:** 추출된 썸네일들을 그리드 형태로 한눈에 보면서, 이미지를 포함/제외하거나 순서를 바꾸고 곧바로 PDF로 내보낼 수 있습니다.
* **반응형의 직관적 갤러리:** 썸네일 크기 조절, 확대 미리보기, 개별 이미지 삭제 기능을 갖춘 반응형 React 갤러리를 제공합니다.
* **빠른 테스트 업로드 흐름:** 개발 중 빠르게 테스트해 볼 수 있도록 `frontend/public/data/test_video.webm`에 영상을 넣고, UI의 '테스트 영상 자동 실행' 버튼을 눌러 과정을 단축할 수 있습니다.
* **사용자 편의성 & 접근성:** 클릭 한 번으로 업로드하고, 진행률 표시와 명확한 오류/로그 안내로 누구나 쉽게 사용할 수 있습니다.

### 전반적인 장점 (Cross-cutting)

* **사용자 중심 UX:** '업로드 → 처리 → 검토 → PDF 생성'으로 이어지는 물 흐르듯 자연스러운 워크플로우 덕분에 비전문가도 쉽게 다룰 수 있습니다.
* **성능을 고려한 설계:** 불필요한 프레임 분석을 건너뛰고, 캐싱 및 백그라운드 작업 분리를 통해 일반적인 데스크탑 환경에서도 쾌적하게 동작합니다.
* **확장 가능한 아키텍처:** 현재는 개인용/로컬 환경을 위해 SQLite + BackgroundTasks를 사용하지만, 다중 사용자 환경을 위한 메시지 브로커, 작업 큐, RDBMS 구조로 손쉽게 확장할 수 있습니다.
* **프라이버시 존중:** 모든 처리와 결과물 생성이 기본적으로 사용자의 로컬 환경에서 이루어지므로, 민감한 영상 데이터가 외부로 유출될 걱정이 없습니다.
* **개발자 친화적:** 깔끔한 모듈 구조(`app/api`, `app/services`, `app/core`, `frontend/src`)로 기능을 추가하거나 코드를 유지보수하기가 매우 쉽습니다.

## 👥 추천 대상

* **뮤지션 & 연주자:** 오프라인이나 인쇄된 악보를 펴놓고 맘 편히 연습하고 싶은 모든 분들.
* **취미 생활자 & 교육자:** 연주 영상에서 교육용 자료나 개인 연습용 출력물을 빠르게 만들고 싶은 분들.

> 💡 **참고:** 현재 아키텍처(SQLite + FastAPI BackgroundTasks)는 개인 및 로컬 환경에 최적화되어 있습니다. 여러 사용자가 함께 쓰는 공개형 서비스로 배포하려면 메시지 브로커(Redis), 작업 큐(Celery), RDBMS(Postgres) 등을 추가로 도입하는 것을 권장합니다.

## 🚀 실행 방법 (단계별 가이드)

여러분의 PC에서 프로젝트를 실행하기 위한 아주 간단한 단계별 가이드입니다.

### 사전 준비 (Prerequisites)

* `Python 3.8+`
* `Node.js`
* `uv` (엄청나게 빠른 파이썬 패키지 설치 및 관리 도구)

### 1. 저장소 클론하기 (Clone the repository)

```bash
git clone https://github.com/yourusername/sheet-music-extractor.git
cd sheet-music-extractor

```

### 2. 백엔드 설정 (FastAPI)

```bash
# 백엔드 폴더로 이동합니다.
cd backend

# uv를 사용하여 가상 환경을 생성합니다.
uv venv

# 가상 환경을 활성화합니다. (Windows)
.venv\Scripts\activate
# 가상 환경을 활성화합니다. (Mac/Linux)
# source .venv/bin/activate

# (권장) uv를 사용하여 의존성 패키지들을 동기화(설치)합니다.
uv sync

# (대안) requirements.txt를 통해 설치할 수도 있습니다.
uv pip install -r requirements.txt
# 또는
python -m pip install -r requirements.txt

# 백엔드 서버를 실행합니다.
uv run uvicorn app.main:app --reload

```

백엔드 API 서버는 `http://localhost:8000` 에서 실행됩니다.

**💡 문제 해결 팁 (의존성 패키지 설치가 실패할 경우):**

* 패키지 설치 전 pip/setuptools/wheel을 먼저 업데이트해 보세요.
```bash
python -m pip install -U pip setuptools wheel

```


* 만약 `uv` 명령어가 제대로 작동하지 않는다면 전역으로 설치하거나 기본 `pip`를 사용하세요.
```bash
python -m pip install --user uv
# 또는 pipx 사용: pipx install uv

```


* **Windows에서 OpenCV 관련 문제가 발생할 경우:**
* 먼저 `pip install opencv-python-headless`로 설치해 보세요. 그래도 안 되면 `pip install opencv-python`을 시도하거나 conda 환경에서 `conda install -c conda-forge opencv`로 설치하세요.
* 빌드(컴파일) 에러가 난다면 "Visual Studio Build Tools(C++)"를 설치하거나 conda를 통해 사전 빌드된 휠(Wheel)을 사용하세요.


* Pillow 등 컴파일이 필요한 패키지 빌드에 실패한다면, 시스템에 빌드 도구가 설치되어 있는지 확인하세요 (Windows: Visual C++ Build Tools, macOS: Xcode command line tools, Linux: build-essential).
* "no matching distribution(일치하는 배포판 없음)" 에러나 휠 호환성 문제가 생기면 `pip`를 업그레이드하거나 파이썬 버전이 3.8 이상인지 확인하세요.
* 최후의 수단으로는 `python -m venv .venv` 명령어로 기본 가상 환경을 만들고 평범한 `pip`로 패키지들을 설치해 보세요.

어떤 에러가 발생했는지 터미널 메시지를 복사해서 알려주시면, 가장 확실한 해결 방법을 찾아드릴 수 있어요!

### 3. 프론트엔드 설정 (React + Vite)

새로운 터미널 창을 열고 프로젝트 최상위 경로에서 다음을 실행합니다.

```bash
# 프론트엔드 폴더로 이동합니다.
cd frontend

# Node 의존성 패키지들을 설치합니다.
npm install

# 개발 서버를 시작합니다.
npm run dev

```

브라우저에서 `http://localhost:5173` 에 접속하면 멋진 UI를 만나보실 수 있습니다!
